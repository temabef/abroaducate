# Legacy Freeze Map

Date: 2026-03-26

## Rule

The current platform is frozen as legacy during the pivot. No new feature work should be added to legacy surfaces except break-fix support if required for local continuity.

## Reusable Foundation

- authentication and session handling
- existing billing and Stripe infrastructure
- account and subscription plumbing
- selected unified profile foundations
- selected admin infrastructure
- selected country/university data foundations where helpful for Germany-first affordability

## Legacy But Preserved

- generic scholarships browsing
- broad university matching
- SOP / cover letter / personal statement flows
- old dashboard
- newsletter and reminders
- visa interview practice
- test prep
- cold email tooling
- broad tools hub
- old homepage messaging

## Retired From The New Core

- all-in-one positioning
- commodity AI writing positioning
- broad “study abroad marketplace” framing
- extension-first execution ideas
- features not directly tied to affordable paths, competitiveness, Story Vault, or Application Playbook

## New Boundary

The new core should live under the Copilot flow.

Legacy tools should remain accessible through a clearly labeled boundary such as:

- `Legacy Tools`

No new copilot workflow should depend on legacy route logic unless the reused logic is explicitly treated as shared foundation.
