## Platform context

- **5,107 registered users**, ~8,000 newsletter subscribers
- **2,597 programs** across 9 countries (Germany, France, Italy, Poland, Lithuania, Estonia, Austria, Czechia, Sweden), all with future deadlines
- **679 scholarships**, 19,773 match rows across 1,429 programs (strict country-only matching)

- **Not deployed yet** — local rebuild. Old platform still live at abroaducate.com.

## ✅ What's been built (complete)

### 1. Scholarship matching system
- Strict country-only matching, 19,773 match rows, no broad-region fallbacks
- FundingGuidance component for programs with 0 matches (Lithuania, Czechia, Estonia)
- Admin Match-Health dashboard at `/admin/match-health`

### 2. Credit system
- 3 lifetime free credits on signup (created via `/api/send-welcome-email` endpoint)
- `spend_credits` RPC handles all deductions atomically
- Low-credit warning email fires at 1 credit remaining (fixed to use `session.user.email` — no longer requires admin client)
- Stripe credit purchases wired end-to-end and QA'd

### 3. Email system (Customer.io, EU region)
- **Welcome email** — fires on signup via `/api/send-welcome-email` (works with email confirmation disabled)
- **Credit low warning** — fires from all 5 generation endpoints when balance hits 1
- **Scholarship digest** — redesigned to match new platform (dark navy header, orange accents, no emoji). Newsletter deduplication: registered users skipped from newsletter send to prevent double-send.
- **Deadline reminder** — redesigned to match new platform. No emoji in subject lines.
- **Test mode**: pass `{"test_email":"your@email.com"}` to `/api/cron/send-reminders` to send only to that address
- DNS verified, `hello@abroaducate.com` live via Zoho Mail

### 4. Pricing page
- Paystack tab removed — Stripe only
- Action costs updated to reflect what platform actually does (removed "CV Translation to Europass", "Scholarship Eligibility Scan"; added Personal Statement)
- Credit pack features corrected (accurate math, "Credits never expire" added)

### 5. Academic CV
- Stub endpoint removed from user-facing flow
- `/academic-cv` replaced with CV Templates & Guidance page (Europass, Overleaf LaTeX, Harvard OCS templates)
- All cross-sell links updated from "Academic CV" to "CV Templates"

### 6. Document generation credit wiring
- SOP, Cover Letter, Personal Statement all now use `spend_credits` RPC (were using legacy monthly counter — 0 credits charged)
- Cover Letter missing `OPENAI_API_KEY` import fixed
- Win Strategy and Scholarship Strategy already correct

### 7. Scholarship page strategy section
- `{:else}` inside `<span>` was causing Svelte parser to misread block depth — fixed by extracting strategy section into `ScholarshipStrategy.svelte` component
- `isPremium` gate removed — "Reveal Strategy" button now always visible for logged-in users with credits
- Strategy fires `spend_credits` server-side; billing modal only shows on actual 402 response

### 8. User profile creation on signup
- `/api/send-welcome-email` now creates `user_profiles` row with `credits = 3` for new users
- Fixes `spend_credits` RPC returning false for new users (no profile row = null credits = RPC returns false)

### 9. Newsletter subscriber cleanup
- 23 fake `@fakeuser.com` rows deleted from `newsletter_subscribers`
- 8,341 real active subscribers remain
- Deduplication: newsletter send skips emails already registered on platform

### 10. AdSense removed
- `AdBootstrap` component removed from root layout
- AdSense CSS removed from `app.html`
- No ads will load on the new platform

### 11. Homepage video updated
- Guideless embed updated to new video: `finding-and-applying-to-scholarships-with-abroaducate-QZAcgafe8jdH`

### 12. All previously completed work

- Contact, booking, AI support chat

## Technical notes

- `execute_sql` RPC cannot run DDL — schema changes must be pasted into Supabase SQL Editor manually
- Windows cmd/PowerShell — use scratch `.mjs` files over inline `node -e`
- Faith Rotich's legacy Stripe subscription is on the old platform — do not touch until after deploy
- Newsletter test: `{"test_email":"your@email.com"}` body param sends only to that address
- Cron emails use `send-reminders` endpoint — never trigger without `test_email` param during local dev (hits live database)
